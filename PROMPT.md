# Proxy Anything Front - 前端需求说明

## 一、整体风格

- **设计风格**：简约，科技感
- **配色方案**：黑白灰为主色调，按钮/链接等交互元素可使用少量点缀色
- **移动端适配**：不需要，仅考虑桌面端
- **内容区最大宽度**：1200px，水平居中

## 二、页面列表

| 页面 | 路由 | 说明 |
|------|------|------|
| 登录页 | `/login` | 用户登录入口 |
| 首页 | `/` | 产品介绍与下载 |
| 个人中心 | `/profile` | 展示用户基本信息 |

## 三、公共布局

首页和个人中心共用同一套布局结构（Header + Main + Footer），路由切换时只替换 Main 区域内容。登录页为独立页面，不使用公共布局。

### 3.1 Header

- 布局：两端对齐（flex justify-between）
- 左侧：Logo，当前为纯文字 "Proxy Anything"，后续会替换为图标+文字
- 右侧：用户头像
  - 头像为圆形，使用用户名首字母生成
  - 点击头像展示下拉菜单，菜单项：
    - **个人中心**：跳转到 `/profile`
    - **退出登录**：清除登录状态，跳转到 `/login`

### 3.2 Footer

- 居中显示版权信息
- 内容：`© 2026 Proxy Anything. All rights reserved.`

## 四、登录页

- 独立页面，不使用公共布局
- 居中显示登录表单
- 表单字段：
  - 用户名（必填）
  - 密码（必填）
  - 登录按钮
- 登录成功后跳转到首页 `/`
- 数据来源：Mock 数据

## 五、首页（Main 区域）

- 左右两列布局（各占 50%），使用 `el-row` + `el-col`
- 两列之间仅靠间距区分，无分割线或不同背景色

### 5.1 左列 - Proxy Anything Client

- **标题**：Proxy Anything Client
- **简介**：一句话概述——轻量、易用的跨平台代理客户端
- **功能特性列表**（使用圆点列表，每条含加粗标签 + 说明）：
  - **多平台支持** — 同时提供 Windows 和 Linux 版本，覆盖主流操作系统
  - **加密 / 非加密通道** — 支持加密与非加密两种模式，加密通信需配合 Proxy Anything Server 实现
  - **TCP & UDP 协议** — 全面支持 TCP 和 UDP 协议转发，满足多样化的网络代理场景
- **配置提示**：使用 `el-alert`（type="warning"，不可关闭，带图标）提示"配置文件需要与应用程序放在同一目录下，客户端启动时会自动读取。"
- **系统类型选择 + 下载软件按钮**：使用 `el-select` 下拉框选择系统类型（Windows / Linux），下拉框与下载按钮在同一行显示，两者使用相同的 `size="large"` 保持视觉一致。按钮文字 "下载客户端"，占位链接（`#`），根据当前选择的系统类型下载对应平台的安装包
- **下载配置按钮**：位于上述行的下方，按钮文字 "下载配置"。无系统类型区分，点击直接下载配置文件（占位链接 `#`）

### 5.2 右列 - Proxy Anything Server

- **标题**：Proxy Anything Server
- **简介**：一句话概述——高性能的代理服务端程序，帮助您快速搭建私有代理服务
- **功能特性列表**（排版与 Client 一致）：
  - **多平台支持** — 同时提供 Windows 和 Linux 版本，覆盖主流操作系统
  - **加密通道** — 作为加密通信的服务端，配合 Proxy Anything Client 实现加密数据传输
  - **TCP & UDP 协议** — 全面支持 TCP 和 UDP 协议转发，满足多样化的网络代理场景
- **配置提示**：使用 `el-alert`（type="warning"，不可关闭，带图标）提示"配置文件需要与应用程序放在同一目录下，服务端启动时会自动读取。"
- **系统类型选择 + 下载软件按钮**：同左列，使用 `el-select` 下拉框 + 下载按钮同行布局，`size="large"`。按钮文字 "下载服务端"，占位链接（`#`），根据当前选择的系统类型下载对应平台的安装包
- **下载配置按钮**：位于上述行的下方，按钮文字 "下载配置"。无系统类型区分，点击直接下载配置文件（占位链接 `#`）

### 5.3 卡片样式

- 白色背景，圆角，轻微阴影
- 内部间距充足，整体视觉舒适

## 六、个人中心（Main 区域）

- 当前阶段仅简单展示用户基本信息，后续再扩展
- 展示内容：
  - 用户头像（大尺寸，用户名首字母生成）
  - 用户名
  - 用户 ID
- 使用 Element Plus 的 `el-descriptions` 组件展示信息

## 七、路由与鉴权

- 使用 Vue Router 路由守卫
- 未登录时访问首页或个人中心，自动跳转到 `/login`
- 已登录时访问 `/login`，自动跳转到首页 `/`
- Token 存储在 Pinia store 中，持久化到 localStorage

## 八、数据层

- 当前阶段使用 Mock 数据
- 通过环境变量 `VITE_USE_MOCK=true` 控制
- Mock 接口：
  - 登录接口：接收用户名密码，返回 token 和用户信息
- 所有接口遵循统一响应格式 `ApiResponse<T>`（code/data/msg）
